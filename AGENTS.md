# Repository Guidelines

## 项目结构与模块组织

当前仓库以文档为主，核心内容在 `Doc/`，包含结构规划、技术选型与接口说明。`README.md` 介绍项目愿景与功能方向。`Doc/structure.md` 描述了未来的目录规划，但目前尚未落地为实际代码目录。

## 构建、测试与本地开发命令

当前仓库未提供可执行的构建或测试命令，也没有 `package.json`、`Makefile` 等脚本入口。请勿自行添加命令说明或默认依赖安装；如需新增脚本，请在文档中同步说明用途与运行方式。

## 编码风格与命名约定

文档统一使用 Markdown，句子尽量简短，首次出现的专业词用一句话解释。文档文件名使用小写与短横线，例如 `Doc/api-system.md`。当前没有具体代码缩进或格式化工具要求，新增代码时以简单清晰为主。

## 测试指南

目前没有测试框架与测试目录。若后续引入测试，请先在 `Doc/structure.md` 中补充规划，再明确测试目录、命名方式与运行命令。

## 提交与拉取请求（PR）要求

当前目录不是 Git 仓库，无法总结既有提交习惯。如未来启用 Git，建议提交信息使用“动词 + 目标”的短句；PR 需要描述目标、影响范围与必要截图或接口示例。

## 其他说明

开发环境以 Windows 为准，命令行默认使用 PowerShell。禁止自动安装依赖，涉及安装或环境变更需先与维护者确认。
